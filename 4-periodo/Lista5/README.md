Primeior foi criado o model Livro.mjs utilizando Sequelize, com os campos id, titulo, autor, ano e paginas. O campo id foi definido como inteiro, chave primária e com autoIncrement, permitindo geração automática pelo banco de dados. Também foi configurada a opção timestamps: true, fazendo com que o Sequelize crie automaticamente os campos createdAt e updatedAt para controle de criação e atualização dos registros.

Depois, foi criado o middleware validarLivro.mjs, responsável por aplicar três camadas de validação nos campos titulo, autor, ano e paginas: presença, tipo e regra de negócio. O middleware verifica se os campos foram enviados, se possuem o tipo esperado e se obedecem às regras do enunciado, como tamanho mínimo, faixa válida de ano e quantidade mínima de páginas. Caso alguma validação falhe, a API retorna status 400 com { "erros": [...] }. O middleware foi aplicado nas rotas POST /livros e PUT /livros/:id, garantindo que apenas dados válidos sejam persistidos no banco.

Após isso, foi criado o middleware logger.mjs, responsável por registrar no terminal a data e hora da requisição, o método HTTP, a URL acessada, o status code e o tempo de resposta em milissegundos. Para medir corretamente o tempo e obter o status final da resposta, foi utilizado res.on("finish"), executado ao final do processamento da requisição. O middleware foi aplicado globalmente no app.mjs com app.use(logger), antes de todas as rotas, garantindo que todas as requisições sejam registradas.

E então o arquivo app.mjs foi atualizado para que o servidor só seja iniciado após a conexão com o banco de dados ser validada e as tabelas serem sincronizadas. Para isso, foi criada uma função assíncrona que utiliza sequelize.authenticate() para testar a conexão e sequelize.sync() para sincronizar o model com o banco. Somente após essas duas etapas o app.listen() é executado. Caso ocorra falha na conexão ou sincronização, o erro é exibido no terminal e a aplicação é encerrada com process.exit(1), conforme solicitado.

Por fim, foi criado o arquivo teste.http para validar todos os cenários exigidos pela lista. Nele foram incluídas requisições para listar livros com banco vazio, cadastrar livro válido, testar erros de validação, buscar por ID existente e inexistente, atualizar, deletar e verificar persistência dos dados após reinício do servidor. Esse arquivo permite comprovar, de forma prática, o funcionamento correto da API, das validações e da persistência em SQLite com Sequelize.

Em resumo, a Lista 5 é uma API de livros com persistência real em SQLite, construída com Express + Sequelize, com model estruturado, validação completa, logger profissional, subida segura do servidor e testes documentados em teste.http.
