Foi criado o middleware autenticar.mjs, responsável por verificar se a requisição contém o header x-api-key com o valor "senha123". Caso o header não exista ou esteja incorreto, a API retorna status 401 com { "erros": ["Acesso nao autorizado"] }. Esse middleware foi aplicado apenas nas rotas POST, PUT e DELETE de usuários, enquanto as rotas GET permaneceram públicas. Dessa forma, apenas operações de escrita e remoção exigem autenticação.
