Foi criada a rota POST /transferencias com três middlewares encadeados: validarTransferencia.mjs, responsável por validar presença, tipo e regras dos campos; logger.mjs, responsável por registrar a requisição; e verificarSaldo.mjs, responsável por verificar se o valor da transferência ultrapassa o limite diário de 1000. Caso o valor exceda esse limite, a API retorna status 422 com { "erros": ["Limite diario excedido"] }. O handler final da rota só executa quando todos os middlewares chamam next(), demonstrando o funcionamento correto do encadeamento no Express.
