Foi criada a rota POST /produtos em uma aplicação Express, junto com o middleware validarProduto.mjs. Esse middleware valida os campos nome, preco e estoque de acordo com as regras do enunciado: nome obrigatório, string e com no mínimo 3 caracteres; preço obrigatório, numérico e maior que zero; estoque obrigatório, inteiro e maior ou igual a zero. Quando há erros, a resposta retorna status 400 com um array de mensagens. Quando os dados são válidos, o middleware chama next() e a rota executa normalmente, cadastrando o produto.
