Foi implementado um middleware logger.mjs responsável por registrar apenas requisições que resultam em erro. Para isso, foi utilizado res.on("finish"), permitindo verificar o statusCode final da resposta apenas após o encerramento da requisição. Quando o status é maior ou igual a 400, o middleware exibe no terminal o método HTTP, a URL acessada e o código de status. Requisições bem-sucedidas, como 200 e 201, não são registradas.
